# Задача: Обязательное заполнение поля ДР в ЛК

**Статус:** В работе
**Дата:** 2026-04-26

## Описание

Сделать обязательным заполнение поля "Дата рождения" в ЛК.

## Сценарий

Сайт → ЛК → Настройки профиля → поле "Дата рождения"

## План / Результат

### Логика:

1. **Подсветка пустого поля ДР** — если ДР не заполнено, колонка с датапикером подсвечивается (highlight-бордер)
2. **Поп-ап на 5 сек** — при входе в настройки профиля с пустым ДР выводить поп-ап: _"Заполни дату рождения, чтобы получить бесплатный подарок от команды «Again»"_
3. **Заполнение ДР** — пользователь заполняет дату рождения через датапикер
4. **Сохранение** — пользователь нажимает "Сохранить изменения"
5. **Блокировка поля** — после сохранения поле ДР становится неактивным (disabled), редактирование невозможно
6. **Подарок на ДР** — за 3 дня до ДР пользователь получает промокод (существующая логика `birthday:process`)

## Технические детали

### Бек

- В `ClientController::update_profile()`:
  - Перед обновлением проверить: если `$client->profile->birthday` уже заполнено — убрать `birthday` из данных для обновления (молча игнорировать)
  - Остальные поля сохраняются как обычно
- Валидация `'birthday' => 'nullable|date'` остаётся без изменений

### Существующая инфраструктура ДР

- `BirthdayDiscountCommand` (`birthday:process`) — cron dailyAt 10:00
- За 3 дня до ДР: создаёт промокод, уведомляет (email, telegram, vk, whatsapp)
- За 1 день до окончания: отправляет напоминание
- Через 6 дней: удаляет промокод
